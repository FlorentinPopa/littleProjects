HTML (HyperText Markup Language)

  1. Definitions
    • Este cel care infrumuseteaza CSS-ul; fara HTML css-ul nu ar avea logica
    • Nu este un limbaj de programare; este un limbaj de marcare a textului si de definire a unor tag -uri.
    • Este limbajul default pentru browser; care citeste limbajul :
      • De la stanga la dreapta
      • De sus in jos
      • Si nu se opreste in caz de err (ex. La C++ se opreste si iti spune ca are o err; dar browser-ul nu notifica asta niciodata);
    • The <!DOCTYPE html> tag: type of document the browser is about to show.
      • All of the document structure is made of tags.
      • All of the document structure is made of tags.
      • All of them are placed within the HTML tag (opened with <html> and closed with </html>).
    • NU scriem tags inafara tag-urilor:
      • in afara <html>
      • intre <html> si <head>
      • intre <head> si <body>
      • intre <body> si <html>
    • HTML is not a programming language.
    • It has no logic and no interaction with users.
    • What a browser does, is interpreting a series of nested tags as a structure of nested elements.
    • Each element has its default appearance and behavior.
    • However it can be changed by: attributes, CSS styles, JS.
  2. Elements are nested inside one another:
    • <span> is nested inside <p>
    • <p> is nested inside <body>>, an element can contain other elements or some text.
    • We usually open a HTML elements with <tag> and close with </tag>
    • We must close the inner element before we close the outer one.
    • Exception: empty elements.
    • <br /> - end of line;
    • <img src="..."> - display an image
    • exista tag-uri simple (exp: <br />, <img src=...>) --> unde nu este nevoie de inchidere tag dupa
    • exista si tag-uri duble - unde este nevoie de inchidere post (exp: <p>...</p>
  3. DOM - Document Object Model
    • A browser interprets a bunch of nested HTML tags as a structure of objects.
    • We call it DOM: -->Document Object Model.
    • We can imagine DOM as a tree.
    • Parent elements contain children elements like in a family tree.
  4. HTML areas:
    • HEAD --> introducem informatiile legate de site, cum se vede in zona de tab (ex. Title, samd)
      • A valid HTML document consists of <head> and <body>.
      • The <head> tells us: kind of content, encoding etc.
      • The elements in the <head> section are not being displayed.
      • Contains the <title> of the document.
      • Can contain different metadata (information for browsers).
      • Can contain some additional scripts.
      • Can contain style rules:
    • BODY -->acolo scriem continutul site-ulu
      • The <body> section contains everything to be displayed.
      • <body>: the actual document for the user.
      • <head>: description of the document for the browser.
  5. HTML attributes
    • HTML elements can have attributes.
    • They provide some extra information about features of an element
    • Syntax: attribute="value"
    • Most common HTML attributes:
      • href - specifies the linkâ€™s address,
      • src - specifies source of the content,
      • title - information shown on hover,
      • id, class - identification of the element for CSS, JS etc.
